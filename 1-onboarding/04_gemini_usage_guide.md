# 生成AI・Gemini 活用ガイド (Gemini Usage Guide)

本ワークショップの核となる「生成AI (Gemini) を効果的に活用してマイグレーションとモダナイゼーションを進める手法」を身につけるためのガイドです。

## 1. Google Cloud 上での Gemini へのアクセス方法

ワークショップでは主に以下の2つのインターフェースを通じて Gemini と対話します。

### ① Google Cloud コンソールの Gemini for Google Cloud
Google Cloud Console 画面右上の「Gemini」アイコン（スパークルのアイコン）をクリックすると、チャットペインが開きます。
- **特徴:** Google Cloud のリソースやドキュメントに特化しており、「今見ている画面のエラー原因を教えて」「Cloud Run のデプロイ手順を教えて」といった、GCP環境固有の質問に強みを発揮します。
- **用途:** アーキテクチャの調査、マネージドサービスの使い方、コンソール上のエラー対応など。

### ② IDE用プラグイン: Gemini Code Assist
VS Code などのエディタにインストールできる拡張機能です。
- **特徴:** 開いているファイルやプロジェクト全体のコードベースをコンテキストとして理解し、インラインでのコード補完、チャット画面でのリファクタリング指示、テストコード生成などが可能です。
- **用途:** SFDC (Apex) からのコード移行、PostgreSQL用DDLの生成、Terraform（IaC）の記述補助、アプリケーションのGo/Pythonなどへの書き換え。
- **インストール手順 (VS Codeの場合):**
  1. 拡張機能タブから `Gemini Code Assist` を検索してインストール。
  2. プロンプトに従って Google Cloud アカウントでサインインし、ワークショップ用プロジェクトを選択します。

## 2. 効果的なプロンプトの基本 (プロンプトエンジニアリング)

AIから望む出力を得るためには、プロンプトに十分な「コンテキスト（文脈）」と「役割（Role）」を付与することが重要です。

### 良いプロンプトの要素
1. **Role (役割の指定):** 指示や前提条件を明確にする
   - 例: 「あなたは Google Cloud のデータベース移行エンジニアです。」
2. **Context (背景):** なぜその作業を行うのか、現状どうなっているかを教える
   - 例: 「現在Salesforceで動いているシステムを、AlloyDBに移行したいと考えています。」
3. **Task (依頼事項):** 何をしてほしいかを具体的に書く
   - 例: 「以下のスキーマ定義をPostgreSQLのDDLに変換してください。」
4. **Format (出力形式):** どのようなフォーマットで返すべきか指定する
   - 例: 「出力は Markdown のコードブロック形式とし、ベストプラクティスに基づいた解説もコメントとして含めてください。」

### ワークショップでの活用例

- **データベース移行:**
  > 「添付のSalesforceオブジェクト定義（Account, Contact）をPostgreSQLのDDLに変換してください。主キーはUUIDを利用し、適切な外部キー制約をつけてください。」
- **コードのモダナイゼーション:**
  > 「以下のApexコード（Trigger）と同じビジネスロジックを実行する Node.js (または Go, Python) の関数を書いてください。エラーハンドリングも適切に実装してください。」
- **IaC (Terraform) の作成:**
  > 「Cloud Run と Cloud SQL (PostgreSQL) を連携させるための Terraform コードのひな形を作成してください。VPC Peering の設定も含めてください。」

AI は一度で完璧な答えを出さないこともあります。生成結果をレビューし、不足していれば「〇〇の考慮が漏れているので、追加して修正してください」と**対話（修正サイクル）を重ねる**ことが成功の秘訣です。
