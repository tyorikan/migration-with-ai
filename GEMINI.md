# Migration with AI: SFDC to Google Cloud ワークショッププロジェクト定義

## 1. プロジェクト概要 (Overview)
現在 Salesforce (SFDC) 上で稼働しているアプリケーションを、生成 AI (Gemini 等) のアプローチを活用して Google Cloud へマイグレーションするための、お客様向け実践ワークショップの定義書です。

## 2. ワークショップのゴール (Goal)
Google エンジニアとお客様のコラボレーションを通じて、**「実際の Google Cloud 環境でアプリケーションが動作するまでのフローを、お客様側が明確に理解し描ける状態にすること」** を最終的なゴールとします。

- 実際のビジネスロジックの完全な実装は必須ではありません。
- ただし、抽象的な議論にとどめず、**具体的な設計やドキュメントをワークショップのアウトプットとして出力** します。

## 3. ワークショップの 6 つの柱 (Core Areas)
本ワークショップでは、お客様の要件やフェーズに応じて、以下の6つの内容の検証・構築を実施します。

### ① オンボーディング
- Google Cloud の基礎知識の習得と、検証用環境を構築するための手順や作業を定義する。

### ② データベース移行
- SFDC のデータ・アーキテクチャから、Google Cloud (Cloud SQL, AlloyDB など) への移行。
- AI を活用した SQL 変換と最適なスキーマ移行戦略を検証するために必要な作業を実施する。

### ③ アプリのモダナイズ
- Gemini を中心とした AI 駆動開発を利用し、既存コード（Apex等）の移行やモダンな環境（Cloud Run や GKE）へのモダナイズ・パスを明確に定義する。

### ④ インフラ＆パイプライン
- Terraform を活用した IaC (Infrastructure as Code) によるインフラ構築の推進。
- CI/CD 環境（デプロイメント・パイプライン）の構築要件と手順を策定する。

### ⑤ ドキュメント作成
- Gemini などの生成 AI を積極的に活用し、設計書や移行ドキュメントの自動生成・メンテナンスを行うための作業フローを確立する。

### ⑥ テスト
- テストコードの自動生成、および自動テスト環境（CI 連携等）の構築を行うための手順とコードの手引きを作成する。

## 4. テクノロジースタック (想定環境)
Google Cloud のアプリケーションモダナイゼーションにおけるベストプラクティスに基づき、以下のサービス群の使用を念頭にワークショップを進行します。
- **コンピュート/基盤:** Cloud Run, Google Kubernetes Engine (GKE)
- **データベース:** Cloud Spanner, Cloud SQL
- **AI/ML:** Gemini for Google Cloud, Vertex AI
- **データ分析/メッセージング:** BigQuery, Pub/Sub
