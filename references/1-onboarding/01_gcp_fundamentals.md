# Google Cloud 基礎知識 (Fundamentals)

本ワークショップ「Migration with AI: SFDC to Google Cloud」を進めるにあたり、Google Cloud の基本的な概念と、今回利用する主要サービスの概要を学びます。

## 1. Google Cloud の基本アーキテクチャ

### リソース階層とプロジェクト
Google Cloud の全リソースは**プロジェクト (Project)** という最下層の論理コンテナに属します。プロジェクトには一意の「プロジェクト ID」が割り当てられ、請求や権限管理の基本的なスコープとなります。
- **組織 (Organization)** > **フォルダ (Folder)** > **プロジェクト** の階層構造をとります。

### IAM (Identity and Access Management)
「谁が (Identity)」「どのリソースに対して」「何ができるか (Role)」を柔軟に制御する仕組みです。
- 基本ロール (閲覧者、編集者、オーナー) は範囲が広すぎるため、本番環境では**事前定義ロール (例: Cloud Run 管理者など)**を付与する「最小権限の原則」を推奨します。

### リージョンとゾーン
- **リージョン (Region):** 独立した地理的エリア (例: `asia-northeast1` = 東京)。
- **ゾーン (Zone):** リージョン内にある単一の障害ドメイン (例: `asia-northeast1-a`)。耐障害性を高めるため、複数ゾーンにまたがるリソース配置 (マルチゾーン構成) が推奨されます。

## 2. ワークショップで登場する主要サービス

本ワークショップのモダナイゼーション・アーキテクチャでは以下の主要プロダクトを活用します。

### Compute (コンピュート/基盤)
- **Cloud Run:** コンテナ化された HTTP アプリケーションをインフラ管理なしでスケーラブルに実行できる、フルマネージドなサーバーレス環境です。ゼロスケールにも対応し、コスト最適化に優れています。
- **Google Kubernetes Engine (GKE):** コンテナ化アプリケーションのデプロイ、管理、スケーリングを自動化するマネージド Kubernetes サービスです。より複雑なマイクロサービス構築に向いています。

### Database (データベース)
- **Cloud SQL:** PostgreSQL, MySQL, SQL Server のフルマネージド リレーショナル データベース サービスです。今回は SFDC からの移行先 (PostgreSQL) として採用します。
- **AlloyDB for PostgreSQL:** Google が提供する、標準 PostgreSQL と完全互換性を持ちながら、トランザクション処理と分析処理を高速化した次世代フルマネージド データベースです。

### AI / ML
- **Vertex AI:** MLモデルの開発、デプロイ、管理を行う統合AIプラットフォームです。生成AIモデル (Gemini) に API を通じてアクセスするための基盤となります。
- **Gemini for Google Cloud:** コンソール上でのチャットアシスタントや、IDEのコーディング支援 (Gemini Code Assist) など、開発・運用をあらゆる面でサポートする AI 機能です。

### データ分析・メッセージング (オプション/発展)
- **BigQuery:** ペタバイトスケールのデータを高速に分析できる、サーバーレスのエンタープライズ データウェアハウスです。
- **Pub/Sub:** マイクロサービス間の非同期メッセージングを可能にする、スケーラブルなイベント取り込み / 配信サービスです。
