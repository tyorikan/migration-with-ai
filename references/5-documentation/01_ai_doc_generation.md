# AI によるドキュメント自動生成 (AI-Assisted Documentation Generation)

モダナイゼーションの過程で、移行先のコードベースや構築したインフラリソースに関するドキュメントを作成することは、チーム間のコミュニケーションや今後の保守において非常に重要です。

## Gemini を活用した仕様抽出

既存のレガシーコードや、新しく作成したモダンコード（Go, Terraform 等）を入力として、わかりやすいマークダウン形式のドキュメントを出力させます。

### プロンプト・テンプレート

#### アプリケーションコードの仕様書生成
```markdown
以下のコードは、モダナイズされた新しいマイクロサービスの実装の一部です。
このコードを分析し、開発者向けのAPI仕様書（またはクラス・モジュール仕様書）をMarkdown形式で作成してください。

要件:
- エンドポイント、リクエストパラメータ、レスポンス形式を明確にすること。
- コード内のコメントやエラーハンドリングのロジックから、考えられるエラーケースの一覧を含めること。
- シーケンス図として Mermaid 形式の記述を追加すること。

【対象コード】
(ここにコードを挿入)
```

## インフラストラクチャのドキュメント化

Terraform コードからシステム構成図やパラメータの説明を抽出するのにも Gemini は有効ですが、SFDCからPostgreSQLへのデータ移行におけるマッピング定義書の生成にも非常に役立ちます。

### プロンプト事例（SFDC to PostgreSQL マッピング）

**テンプレート:** 生成結果は [データマッピング仕様書テンプレート](docs/templates/data-mapping-template.md) のフォーマットに合わせてレビューしてください。

```text
以下の Salesforce のオブジェクト定義 JSON と、移行先の PostgreSQL DDL をもとに、
データマッピング仕様書を Markdown 形式で作成してください。

要件:
1. SFDC 側の項目名(API)、データ型、PostgreSQL のカラム名、データ型を表形式で整理する
2. 型変換に伴う注意点を備考欄に記載する（例: Currency → NUMERIC の精度、Picklist → VARCHAR vs CHECK制約）
3. 以下の SFDC 固有概念については、PostgreSQL での推奨実装パターンを明記する:
   - 数式項目 (Formula Field) → Generated Column / View / アプリ層
   - ロールアップ集計 (Roll-Up Summary) → View + 集計関数 / Materialized View
   - レコードタイプ (Record Type) → record_type カラム
   - ポリモーフィック参照 (WhatId 等) → 個別 FK カラム / Junction Table
4. 外部キー制約の ON DELETE 動作を、SFDC のリレーション種別に基づいて設定する:
   - Lookup → ON DELETE SET NULL
   - Master-Detail → ON DELETE CASCADE
5. 推奨インデックスの一覧を含める

【SFDC オブジェクト定義 JSON】
(ここに sample_sfdc_schema.json の内容またはオブジェクト定義を挿入)

【PostgreSQL DDL】
(ここに生成済み DDL を挿入)
```

### 移行影響分析レポート生成プロンプト

```text
以下の SFDC の Apex コード（トリガー / クラス）と、移行先の Go コードを分析し、
移行影響分析レポートを Markdown 形式で作成してください。

要件:
1. 各 Apex トリガーの処理内容と、対応する Go 側の実装状況をマッピング表にする
2. 未移行のビジネスロジックを「移行 TODO」として一覧化する
3. SFDC 固有機能（ワークフロールール、プロセスビルダー等）で
   コードに現れない暗黙のロジックがないか注意喚起する
4. リスク評価（High/Medium/Low）を各項目につける

【Apex コード】
(ここに Apex コードを挿入)

【Go コード】
(ここに Go コードを挿入)
```

### インフラ構成プロンプト事例
```text
以下の Terraform の main.tf と variables.tf の内容を読み込み、Cloud Run と Cloud SQL を用いたインフラ構成の概要ドキュメントを作成してください。
各リソースの目的と、重要な変数の説明をテーブルでまとめてください。
```

## 成果物のレビューと調整

AI によって生成されたドキュメントは、必ずエンジニアがレビューし、プロジェクト特有の背景やビジネス要件のニュアンスが欠落していないかを確認・補完してください。
